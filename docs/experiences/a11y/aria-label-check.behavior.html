The "ariaLabelCheckBehavior" Experience

Overview

  • UX Token: ariaLabelCheckBehavior
  • Description: Contains logic to check the HTML Element for proper aria-label or aria-labeled-by. This experience will produce a console warning when an aria-label is missing. Use when you want to ensure aria-labels are correctly placed.
  • Use Case: This experience is meant for developers and provides reminders for where aria-labels should be added to enhance the user experience for assistive technology users. When developing experiences, this can be a helpful tool to guide those implementing your experiences to make sure they are placing aria-labels at the correct locations in code.
  • Class Definition: AriaLabelCheckBehavior

Demo

Example of aria-label Missing

To see this experience in action, open the console in your browser's developer tools. The experience will log a warning if the element is missing an aria-label. In this demo, you will see that when the aria-label and aria-labelledby is missing on the element, a console warning will be present.

Code:

<button ux="ariaLabelCheckBehavior">
    Button Without aria-label
</button>

Example of aria-label Present

To see this experience in action, open the console in your browser's developer tools. The experience will log a warning if the element is missing an aria-label. In this demo, you will see that when the aria-label is present on the element, there will be no console warning.

Code:

<button ux="ariaLabelCheckBehavior"
    aria-label="opens modal">
    Button With aria-label
</button>

Example of aria-labeledby Present

To see this experience in action, open the console in your browser's developer tools. The experience will log a warning if the element is missing an aria-label. In this demo, you will see that when the aria-labeledby is present on the element, there will be no console warning.

Code:

<button ux="ariaLabelCheckBehavior"
    aria-labelby="modalTitle">

API

Methods

  • AriaLabelCheckBehavior::onRefresh() - This is an event hook method method is called when the experience is refreshed. It checks if the element has an aria-label or aria-labelledby attribute and logs a warning if neither is present.
  • AriaLabelCheckBehavior::isAriaLabelPresent() - This method checks if the aria-label or aria-labelledby attribute is present on the element. It returns true if either attribute is present, otherwise false.
